# 🎯 Исправление позиционирования Tour Guide

## ❌ Проблема

При использовании tour на маленьких экранах:
- Подсветка "плавала" и не точно выделяла элементы
- Позиции обновлялись неправильно после скролла
- Рекурсивные вызовы updateTargetPosition при скролле
- Tooltip позиционировался некорректно

---

## ✅ Решение

### 1. **Добавлен флаг `isScrolling`**
```typescript
const [isScrolling, setIsScrolling] = useState(false);
```

**Назначение:**
- Предотвращает повторный автоскролл во время анимации
- Блокирует обновления позиций до завершения скролла
- Избегает рекурсивных вызовов

---

### 2. **Разделены режимы обновления позиций**
```typescript
updateTargetPosition(shouldScroll = true)
```

**Два режима:**
- `shouldScroll: true` - При смене шага (автоскролл + позиционирование)
- `shouldScroll: false` - При ручном скролле/resize (только позиционирование)

---

### 3. **Увеличены отступы для проверки видимости**
```typescript
// Было:
rect.top >= 0 && rect.bottom <= window.innerHeight

// Стало:
rect.top >= 80 && rect.bottom <= window.innerHeight - 80
```

**Преимущества:**
- Элемент считается "невидимым" если слишком близко к краю
- Автоскролл срабатывает раньше для лучшего UX
- 80px сверху и снизу - зона комфорта

---

### 4. **Улучшен обработчик скролла**
```typescript
const handleScroll = () => {
  if (!isScrolling) {
    updateTargetPosition(false); // Только обновление, без автоскролла
  }
};

window.addEventListener("scroll", handleScroll, { passive: true });
```

**Улучшения:**
- Проверка флага `isScrolling` перед обновлением
- `passive: true` для лучшей производительности
- Не вызывает повторный автоскролл

---

### 5. **Увеличен таймаут анимации**
```typescript
// Было:
setTimeout(() => { ... }, 600);

// Стало:
setTimeout(() => { 
  // Обновление позиций
  setIsScrolling(false); // Сброс флага
}, 700);
```

**Причины:**
- 700ms достаточно для завершения smooth scroll
- Сброс флага после завершения анимации
- Более плавная работа на медленных устройствах

---

## 🎬 Последовательность работы

### При смене шага tour:

```
1. Пользователь нажимает "Далее"
   ↓
2. currentStep изменяется
   ↓
3. useEffect вызывает updateTargetPosition(true)
   ↓
4. Находит целевой element (querySelector)
   ↓
5. Получает rect = element.getBoundingClientRect()
   ↓
6. Проверяет видимость:
   - top >= 80?
   - bottom <= innerHeight - 80?
   ↓
7. Если НЕ виден:
   a. setIsScrolling(true) ← Блокирует дальнейшие скроллы
   b. Вычисляет scrollToY для центрирования
   c. window.scrollTo({ behavior: 'smooth' })
   d. Через 700ms:
      - Обновляет rect
      - Позиционирует tooltip
      - setIsScrolling(false) ← Разблокирует
   ↓
8. Если виден:
   - Сразу позиционирует tooltip
```

---

### При ручном скролле пользователем:

```
1. Пользователь скроллит колесиком мыши
   ↓
2. Срабатывает handleScroll
   ↓
3. Проверяет: if (!isScrolling)
   ↓
4. Если НЕ идет автоскролл:
   - updateTargetPosition(false) ← Без автоскролла
   - Обновляет rect
   - Перепозиционирует tooltip и подсветку
   ↓
5. Если идет автоскролл (isScrolling = true):
   - Игнорирует событие
   - Ждет завершения анимации
```

## 🎯 Улучшения в деталях

### Проверка видимости с отступами:

```typescript
const isElementVisible = (
  rect.top >= 80 &&              // 80px от верха
  rect.left >= 20 &&             // 20px от левого края
  rect.bottom <= innerHeight - 80 && // 80px от низа
  rect.right <= innerWidth - 20  // 20px от правого края
);
```

**Визуализация:**
```
┌─────────────────────────────┐
│     80px (зона невидимости)  │ ← Если элемент здесь, скроллим
├─────────────────────────────┤
│                             │
│   ВИДИМАЯ ЗОНА              │ ← Элемент виден, не скроллим
│                             │
├─────────────────────────────┤

└─────────────────────────────┘
```

## 🔧 Технические детали

### Флаг isScrolling:
```typescript
// Устанавливается перед автоскроллом
setIsScrolling(true);

// Блокирует повторные вызовы
if (!isScrolling) {
  updateTargetPosition(false);
}

// Сбрасывается после завершения
setTimeout(() => {
  setIsScrolling(false);
}, 700);
```

### Параметр shouldScroll:
```typescript
// При смене шага - с автоскроллом
updateTargetPosition(true);

// При ручном скролле - без автоскролла
updateTargetPosition(false);
```
