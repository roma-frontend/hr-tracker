# 🧪 ТЕСТИРОВАНИЕ UX УЛУЧШЕНИЙ

## ✅ СЕРВЕР ЗАПУЩЕН!

Откройте **http://localhost:3000** в браузере.

---

## 📋 ЧЕКЛИСТ ДЛЯ ПРОВЕРКИ

### **1. Command Palette (Ctrl+K)** 🔍

**Где:** Любая страница dashboard

**Шаги:**
1. Войдите в систему (если не вошли)
2. Откройте http://localhost:3000/dashboard
3. Нажмите **Ctrl+K** (или Cmd+K на Mac)
4. Введите "отпуск" или "leave"
5. Нажмите стрелку ↓ для навигации
6. Нажмите Enter для перехода

**Ожидаемый результат:**
- [ ] Модальное окно поиска открывается
- [ ] Круглая кнопка с иконкой Command справа внизу
- [ ] 15+ команд в списке
- [ ] Навигация клавиатурой работает (↑↓)
- [ ] Enter переходит на страницу

---

### **2. Dashboard Tour** 🎯

**Где:** Страница dashboard

**Шаги:**
1. Откройте http://localhost:3000/dashboard
2. Если тур не появился автоматически:
   - Откройте консоль (F12)
   - Введите: `localStorage.removeItem("dashboard_tour_seen")`
   - Обновите страницу (F5)

**Ожидаемый результат:**
- [ ] Тур появляется через 1 секунду
- [ ] 7 шагов с иконками
- [ ] Подсветка элементов (stats, quick actions)
- [ ] Кнопки "Назад", "Далее", "Пропустить"
- [ ] Прогресс (1/7, 2/7, etc.)
- [ ] Затемнение фона

### **3. Quick Actions** ⚡

**Где:** Страница dashboard

**Шаги:**
1. Откройте http://localhost:3000/dashboard
2. Прокрутите вниз после карточек статистики
3. Найдите секцию "⚡ Быстрые действия"
4. Нажмите на любую кнопку (например, "Подать заявку на отпуск")

**Ожидаемый результат:**
- [ ] 8 цветных кнопок в grid (2x4 или 4x2)
- [ ] При наведении — увеличение (scale 1.05)
- [ ] Toast уведомление "Открываю..."
- [ ] Переход на соответствующую страницу

**Кнопки:**
- [ ] Подать заявку на отпуск (синяя)
- [ ] Отметиться (зеленая)
- [ ] Чат (фиолетовая)
- [ ] Задачи (оранжевая)
- [ ] Подтверждения (индиго, для менеджеров)
- [ ] Аналитика (розовая, для менеджеров)
- [ ] Сотрудники (бирюзовая, для админов)
- [ ] Настройки (серая, для админов)

### **4. Leave Request Wizard** 📝

**Где:** Страница отпусков

**Шаги:**
1. Откройте http://localhost:3000/leaves
2. Нажмите зеленую кнопку "+ New Request"
3. Пройдите все 3 шага:
   - **Шаг 1:** Выберите тип отпуска и даты
   - **Шаг 2:** Проверьте баланс, введите причину
   - **Шаг 3:** Подтвердите заявку

**Ожидаемый результат:**
- [ ] Wizard открывается (не старый modal!)
- [ ] Индикатор прогресса (3 шага)
- [ ] Шаг 1: 5 типов отпуска, календарь
- [ ] Шаг 1: Подсчет дней автоматически
- [ ] Шаг 2: Баланс до/после
- [ ] Шаг 2: Проверка на достаточность дней
- [ ] Шаг 3: Сводка всех данных
- [ ] Шаг 3: Кнопка "Отправить заявку"

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### **Проблема 1: Command Palette не виден**

**Решение:**
```bash
# Проверьте консоль браузера на ошибки
F12 → Console → Проверьте на ошибки

# Проверьте что компонент в layout
Откройте src/app/(dashboard)/layout.tsx
Убедитесь что есть: <CommandPalette />
```

### **Проблема 2: Dashboard Tour не появляется**

**Решение:**
```javascript
// В консоли браузера (F12):
localStorage.removeItem("dashboard_tour_seen")
location.reload()
```

### **Проблема 3: Quick Actions не видны**

**Решение:**
```bash
# Проверьте консоль на ошибки импорта
F12 → Console → Проверьте на "Cannot find module"

# Убедитесь что компонент импортирован:
Откройте src/components/dashboard/DashboardClient.tsx
Найдите: import { QuickActions } from ...
```

### **Проблема 4: Wizard не открывается**

**Решение:**
```bash
# Проверьте что кнопка вызывает wizard:
Откройте src/components/leaves/LeavesClient.tsx
Найдите: onClick={() => setWizardOpen(true)}

# Проверьте что Wizard рендерится:
Найдите: {wizardOpen && <LeaveRequestWizard ... />}
```
