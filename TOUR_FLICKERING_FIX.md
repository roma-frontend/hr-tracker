# 🔧 Исправление мигания Tour Guide

## ❌ Проблема

**Симптомы:**
- Модальное окно tooltip постоянно мигает/моргает
- Кажется, что оно обновляется постоянно
- Особенно заметно на больших экранах
- Дрожание подсветки элементов

**Причина:**
- Отсутствие debounce для событий scroll и resize
- Слишком частые пересчёты позиций (каждый пиксель скролла)
- Подсветка отрисовывалась до готовности позиционирования
- Мгновенные вызовы updateTargetPosition на каждое событие

---

## ✅ Решение

### 1. **Debounce для scroll событий**
```typescript
let scrollTimeout: NodeJS.Timeout;

const handleScroll = () => {
  if (isScrolling) return;
  
  // Debounce - ждём 100ms после последнего скролла
  clearTimeout(scrollTimeout);
  scrollTimeout = setTimeout(() => {
    updateTargetPosition(false);
  }, 100);
};
```

**Эффект:** 
- Обновление позиций только после остановки скролла
- Вместо 100 вызовов → 1 вызов
- Нет мигания при скролле

---

### 2. **Debounce для resize событий**
```typescript
let resizeTimeout: NodeJS.Timeout;

const handleResize = () => {
  if (isScrolling) return;
  
  // Debounce - ждём 150ms после последнего resize
  clearTimeout(resizeTimeout);
  resizeTimeout = setTimeout(() => {
    updateTargetPosition(false);
  }, 150);
};
```

**Эффект:**
- Обновление только после завершения ресайза
- Плавная работа при изменении размера окна
- Нет дрожания на больших экранах

---

### 3. **Подсветка только после позиционирования**
```typescript
// Было:
{targetRect && step.highlight !== false && (
  <motion.div ... />
)}

// Стало:
{targetRect && step.highlight !== false && isPositioned && (
  <motion.div ... />
)}
```

**Эффект:**
- Подсветка появляется только после готовности
- Нет мигания при первом рендере
- Синхронизация с tooltip

---

### 4. **Очистка таймаутов**
```typescript
return () => {
  clearTimeout(resizeTimeout);
  clearTimeout(scrollTimeout);
  window.removeEventListener("resize", handleResize);
  window.removeEventListener("scroll", handleScroll);
};
```

**Эффект:**
- Предотвращение утечек памяти
- Корректное завершение при unmount
- Нет "висячих" таймеров

---

## 🎬 Как работает debounce

### До (без debounce):
```
Скролл на 100px:
↓
Событие scroll #1 → updateTargetPosition()
Событие scroll #2 → updateTargetPosition()
Событие scroll #3 → updateTargetPosition()
... (еще 97 раз)
Событие scroll #100 → updateTargetPosition()

Результат: 100 пересчётов = МИГАНИЕ
```

### После (с debounce):
```
Скролл на 100px:
↓
Событие scroll #1 → Таймер 100ms
Событие scroll #2 → Сброс таймера, новый таймер 100ms
Событие scroll #3 → Сброс таймера, новый таймер 100ms
... (еще 97 раз)
Событие scroll #100 → Сброс таймера, новый таймер 100ms
↓
(Скролл остановился)
↓
Через 100ms → updateTargetPosition()

Результат: 1 пересчёт = НЕТ МИГАНИЯ
```

## 🎯 Технические детали

### Таймауты debounce:
```typescript
scroll:  100ms  // Быстрая реакция на остановку скролла
resize:  150ms  // Чуть дольше для стабилизации размера
```

### Почему эти значения:
- **100ms для scroll:** Оптимальный баланс между отзывчивостью и производительностью
- **150ms для resize:** Окно браузера стабилизируется после resize за ~100-150ms

### Логика проверок:
```typescript
if (isScrolling) return;  // Не обновлять во время автоскролла
clearTimeout(timeout);     // Сбросить предыдущий таймер
setTimeout(...)           // Новый таймер с задержкой
```
