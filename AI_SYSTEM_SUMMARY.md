## 🔐 Система безопасности

### Многоуровневая проверка:
1. **Frontend** - `aiAssistant.ts` проверяет разрешения перед отображением
2. **API** - `route.ts` проверяет роль перед построением промпта
3. **AI Prompt** - содержит явные инструкции о разрешениях роли
4. **Backend** - Convex функции проверяют права на уровне БД

### Примеры защиты:
```typescript
// Employee пытается получить доступ к admin функции
detectIntent("show all organizations", "employee") 
// → null (не найдено в доступных capabilities)

// Admin пытается получить superadmin функцию
hasPermission("admin", ["superadmin"]) 
// → false

// AI промпт для employee содержит:
"SCOPE: You can only access your own personal data."
```

## 🛠️ Как это работает

### Сценарий 1: Голосовая навигация
```
1. Пользователь нажимает 🎤 в чате
2. Speech Recognition API начинает слушать
3. Пользователь говорит: "Покажи календарь"
4. Текст появляется в поле ввода
5. Через 1 сек паузы → автоотправка
6. API получает запрос с userId и текстом
7. detectIntent("Покажи календарь", userRole)
   → находит capability "view_calendar" → /calendar
8. AI получает подсказку: 'Include <NAVIGATE>/calendar</NAVIGATE>'
9. AI отвечает: "Открываю календарь... 📅 <NAVIGATE>/calendar</NAVIGATE>"
10. ChatWidget парсит тег <NAVIGATE>
11. Через 800ms → router.push('/calendar')
12. Чат закрывается
```

### Сценарий 2: Текстовый запрос без навигации
```
1. Пользователь вводит: "Сколько у меня отпуска?"
2. detectIntent() → null (это не навигация, а запрос данных)
3. AI получает полный контекст:
   - Роль пользователя
   - Балансы отпусков
   - Статистика
4. AI отвечает с точными данными:
   "У вас осталось:
    📅 Оплачиваемый отпуск: 15 дней
    🤒 Больничный: 7 дней
    👨‍👩‍👧 Семейный: 5 дней"
5. Добавляются follow-up подсказки
```

### Сценарий 3: Попытка доступа без прав
```
1. Employee вводит: "Покажи аналитику"
2. detectIntent("Покажи аналитику", "employee")
   → находит "view_analytics"
   → requiredRole: ["admin", "superadmin"]
   → hasPermission("employee", ["admin", "superadmin"]) → false
   → return null (capability не доступна)
3. AI получает промпт с ограничениями employee:
   "SCOPE: You can only access your own personal data."
4. AI отвечает:
   "❌ К сожалению, аналитика доступна только администраторам.
   Вы можете посмотреть свою личную статистику в профиле."
```
