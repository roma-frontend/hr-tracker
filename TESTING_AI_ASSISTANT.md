# 🧪 Руководство по тестированию AI ассистента

## 📋 Чеклист для тестирования

### 1. Тестирование ролей и разрешений

#### Тест 1: Сотрудник (Employee)
**Цель:** Проверить, что сотрудник видит только свои данные и ограниченные функции

**Шаги:**
1. Войдите как сотрудник (не admin)
2. Откройте AI ассистент (иконка ✨)
3. Проверьте начальные подсказки - должны быть:
   - "Покажи мой календарь"
   - "Сколько у меня осталось отпуска?"
   - "Кто сегодня на работе?"
   - "Мои задачи"

**Команды для проверки:**
```
✅ "покажи календарь" → должен открыть /calendar
✅ "мои отпуска" → должен открыть /leaves
✅ "мои задачи" → должен открыть /tasks
✅ "мой профиль" → должен открыть /profile
❌ "список сотрудников" → должен сказать "нет доступа"
❌ "аналитика" → должен сказать "доступно только админам"
❌ "все организации" → должен сказать "нет доступа"
```

#### Тест 2: Супервайзер (Supervisor)
**Шаги:**
1. Войдите как супервайзер
2. Откройте AI ассистент
3. Проверьте подсказки - должны включать команды управления командой

**Команды для проверки:**
```
✅ Все команды сотрудника
✅ "заявки на отпуск" → /approvals
✅ "посещаемость команды" → /attendance
❌ "список всех сотрудников" → ограниченный доступ
❌ "аналитика организации" → нет прав
```

#### Тест 3: Администратор (Admin)
**Шаги:**
1. Войдите как администратор
2. Откройте AI ассистент
3. Проверьте расширенные функции

**Команды для проверки:**
```
✅ Все команды сотрудника + супервайзера
✅ "список сотрудников" → /employees
✅ "аналитика" → /analytics
✅ "отчеты" → /reports
✅ "настройки" → /settings
✅ "заявки на присоединение" → /join-requests
❌ "все организации" → только для superadmin
❌ "stripe подписки" → только для superadmin
```

### 2. Тестирование голосового управления

#### Тест 5: Голосовая навигация (русский)
**Шаги:**
1. Откройте чат AI
2. Нажмите на иконку микрофона 🎤
3. Произнесите команды:

```
🎤 "Покажи календарь"
→ Ожидаемо: Текст появляется в поле ввода, через 1 сек автоотправка, AI отвечает и открывает /calendar

🎤 "Открой мои задачи"
→ Ожидаемо: Открывает /tasks

🎤 "Мой профиль"
→ Ожидаемо: Открывает /profile

🎤 "Список сотрудников" (только для admin)
→ Ожидаемо: Открывает /employees или "нет доступа"
```

#### Тест 6: Голосовая навигация (английский)
```
🎤 "Show calendar"
→ /calendar

🎤 "Open my tasks"
→ /tasks

🎤 "My profile"
→ /profile

🎤 "Show analytics" (admin only)
→ /analytics или "no access"
```

#### Тест 7: Голосовое бронирование отпуска
```
🎤 "Хочу взять отпуск с 10 по 15 марта"
→ Ожидаемо: AI создает карточку бронирования с датами и кнопкой подтверждения

🎤 "I want vacation from March 10 to March 15"
→ Ожидаемо: Аналогично на английском
```

### 4. Тестирование навигации

#### Тест 10: Автоматическая навигация
**Процесс:**
1. Отправьте команду навигации
2. Проверьте, что AI:
   - Отвечает подтверждением ("Открываю календарь... 📅")
   - Через 800ms автоматически перенаправляет
   - Закрывает чат после навигации

**Примеры:**
```
"покажи календарь" 
→ AI: "Открываю календарь... 📅"
→ Через 0.8 сек: переход на /calendar
→ Чат закрывается

"аналитика" (для admin)
→ AI: "Открываю аналитику... 📊"
→ Переход на /analytics
```

#### Тест 11: Отказ в навигации (нет прав)
```
Employee пытается: "все организации"
→ AI: "❌ Эта функция доступна только для супер-администратора. 
        Вы можете посмотреть информацию о вашей организации в профиле."
→ Навигация НЕ происходит
```

## 🎯 Быстрый тест (5 минут)

Для быстрой проверки работоспособности:

1. **Откройте чат AI** (✨)
2. **Проверьте подсказки** - соответствуют роли?
3. **Голосовая команда:** 🎤 "Покажи календарь" → открылся /calendar?
4. **Текстовая команда:** "мои задачи" → открылся /tasks?
5. **Проверка прав:** (для employee) "аналитика" → отказ в доступе?

✅ Если все 5 пунктов работают - система функционирует корректно!

### Включение debug логов:

Откройте консоль браузера (F12) и следите за:
```
🤖 AI Chat request received
🎯 Voice navigation: /calendar

```

### Частые проблемы:

**Проблема:** Голосовой ввод не работает
→ **Решение:** Проверьте разрешения микрофона в браузере

**Проблема:** AI не распознает команду
→ **Решение:** Проверьте keywords в `src/lib/aiAssistant.ts`

**Проблема:** Навигация не происходит
→ **Решение:** Проверьте роль пользователя и permissions
